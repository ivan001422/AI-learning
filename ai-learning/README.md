## Глава 1. Введение в машинное обучение

### Зачем нужен ML
- Традиционные системы на основе правил «если-то» плохо масштабируются, требуют экспертных знаний и не работают в задачах, где невозможно формализовать правила (например, распознавание лиц).
- ML автоматически извлекает закономерности из данных, обобщая известные примеры.

### Типы задач и их примеры

| Тип | Характеристика | Подтипы | Примеры |
|-----|----------------|---------|---------|
| **Обучение с учителем** | Есть входные данные (X) и правильные ответы (y). Модель учится предсказывать y для новых X. | **Классификация** – дискретные метки (бинарная, мультиклассовая).<br>**Регрессия** – непрерывное значение. | Спам/не спам, сорт ириса, цена дома, доход, температура. |
| **Обучение без учителя** | Есть только X, ответов нет. Модель ищет скрытую структуру. | **Кластеризация** – разбиение на группы.<br>**Преобразование данных** – снижение размерности, визуализация, выделение тем. | Сегментация клиентов, тематическое моделирование, обнаружение аномалий. |

### Представление данных (табличный вид)
- **Пример (sample)** – строка (один объект).
- **Признак (feature)** – столбец (свойство).
- Обозначения в `scikit-learn`:
  - `X` – двумерный массив (матрица) признаков, форма `(n_samples, n_features)`.
  - `y` – одномерный массив (вектор) целевых меток (для классификации) или значений (для регрессии).

### Стек инструментов: Python, NumPy, SciPy, matplotlib, pandas, scikit-learn, mglearn
- **Python** – основной язык, рекомендуется Python 3.
- **NumPy** – основа для хранения и обработки массивов. В `scikit-learn` все данные передаются как массивы NumPy или разреженные матрицы SciPy.
- **SciPy** – содержит разреженные матрицы (`scipy.sparse`), экономящие память при большом количестве нулей.
- **matplotlib** – визуализация (графики, гистограммы, диаграммы рассеяния).
- **pandas** – таблицы (DataFrame), удобен для загрузки, очистки и анализа данных.
- **scikit-learn** – библиотека ML с единым API для разных алгоритмов.
- - **mglearn** – упрощает построение графиков и создание синтетических наборов данных (например, `make_forge`, `make_wave` и др.).

### Первый пример: классификация ирисов (Iris) – пошагово

**Задача**: по четырём измерениям (длина/ширина чашелистика и лепестка) определить сорт ириса (setosa, versicolor, virginica).

**Данные** – встроенный набор `load_iris()` из `sklearn.datasets`. Это классический набор данных, содержащий 150 образцов ирисов трёх сортов (по 50 каждого). Признаки: длина и ширина чашелистика и лепестка (в см). Целевая переменная – номер сорта (0, 1, 2).
```python
from sklearn.datasets import load_iris
iris = load_iris()
print(iris.keys())  # dict_keys(['data', 'target', 'target_names', 'feature_names', 'DESCR'])
print(iris.data.shape)      # (150, 4)
print(iris.target.shape)    # (150,)
print(iris.target_names)    # ['setosa' 'versicolor' 'virginica']
```

**Разбиение на обучающую и тестовую выборки**:
```python
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(
    iris['data'], iris['target'], random_state=0)
```
- `random_state` – фиксирует случайное перемешивание для воспроизводимости.
- По умолчанию тестовый набор – 25% данных.
- Важно: **тестовый набор не должен использоваться в процессе обучения** – только для итоговой оценки.

**Визуализация** (разведочный анализ):
```python
import pandas as pd
import mglearn
iris_dataframe = pd.DataFrame(X_train, columns=iris.feature_names)
grr = pd.scatter_matrix(iris_dataframe, c=y_train, figsize=(15,15), marker='o',
                        hist_kwds={'bins': 20}, s=60, alpha=.8, cmap=mglearn.cm3)
```
Матрица диаграмм рассеяния показывает, что классы хорошо разделяются по некоторым признакам.

**Построение модели k-ближайших соседей**:
```python
from sklearn.neighbors import KNeighborsClassifier
knn = KNeighborsClassifier(n_neighbors=1)   # только один ближайший сосед
knn.fit(X_train, y_train)
```

**Прогноз для нового цветка**:
```python
X_new = np.array([[5, 2.9, 1, 0.2]])   # форма должна быть (1, 4)
prediction = knn.predict(X_new)
print(prediction)                      # [0] -> setosa
print(iris['target_names'][prediction]) # ['setosa']
```

**Оценка на тестовом наборе**:
```python
y_pred = knn.predict(X_test)
accuracy = np.mean(y_pred == y_test)   # ≈ 0.97
# или
accuracy = knn.score(X_test, y_test)   # ≈ 0.97
```
Правильность (accuracy) – доля правильных ответов. В примере 97% – отличный результат.

**Итоговый код** (весь цикл):
```python
X_train, X_test, y_train, y_test = train_test_split(
    iris_dataset['data'], iris_dataset['target'], random_state=0)
knn = KNeighborsClassifier(n_neighbors=1)
knn.fit(X_train, y_train)
print("Accuracy: {:.2f}".format(knn.score(X_test, y_test)))
```

## Глава 2. Методы машинного обучения с учителем

### Классификация и регрессия – определения
- **Классификация** – предсказание дискретной метки. Бинарная – два класса (положительный/отрицательный), мультиклассовая – более двух.
- **Регрессия** – предсказание непрерывного значения. Примеры: цена, доход, температура.

### Обобщающая способность, переобучение и недообучение
- **Обобщение** – способность модели делать правильные прогнозы на новых, невиданных данных. Это главная цель.
- **Недообучение** – модель слишком простая, не может выявить закономерности в обучающих данных. Низкая точность как на обучении, так и на тесте.
- **Переобучение** – модель слишком сложная, «запоминает» обучающие данные, включая шум и выбросы. Высокая точность на обучении, но низкая на тесте.
- **Компромисс** – существует оптимальная сложность модели, которая даёт максимальную обобщающую способность. Увеличение объёма данных позволяет строить более сложные модели без переобучения.

### Связь сложности модели и размера данных
- Чем больше разнообразных данных, тем сложнее модель можно обучить без переобучения.
- Простое копирование или дублирование данных не помогает – нужна новая информация.
- В реальных проектах сбор большего количества данных часто даёт больший прирост качества, чем настройка алгоритма.

---

### Алгоритмы обучения с учителем (подробно)

#### k-ближайших соседей (k-NN)

**Классификация**:
- Модель хранит все обучающие точки.
- Для прогноза для новой точки находятся `k` ближайших соседей по евклидову расстоянию (или другой метрике) и проводится голосование.
- Граница решений становится более изрезанной при малом `k` (высокая сложность) и более гладкой при большом `k` (низкая сложность).
- Пример на датасете **cancer** (набор данных о раке молочной железы из `sklearn.datasets.load_breast_cancer` – 569 образцов с 30 числовыми признаками, полученными из изображений клеток, задача бинарной классификации: доброкачественная/злокачественная опухоль): при `k=1` точность на обучении 100%, на тесте ~0.88; при `k=6` точность на обучении ~0.93, на тесте ~0.94. Оптимальный `k` ≈ 6–8.

**Регрессия (`KNeighborsRegressor`)**:
- Прогноз – среднее (или медианное) значение целевой переменной среди `k` соседей.
- На наборе **wave** (синтетический набор данных из `mglearn.datasets.make_wave` – 40 точек с одним признаком и непрерывным целевым значением, смоделированным как волна) видно, что `k=1` даёт сильные колебания, `k=9` – сглаживание.

**Важные параметры**:
- `n_neighbors` – число соседей (основной).
- `metric` – метрика расстояния (по умолчанию `'minkowski'` с `p=2`, т.е. евклидово).

**Преимущества**:
- Простая интерпретация.
- Хорош как базовый алгоритм (baseline).

**Недостатки**:
- Медленно предсказывает для больших наборов данных (O(n_samples)).
- «Проклятие размерности» – при большом количестве признаков расстояния становятся неинформативными.
- Требует масштабирования признаков (т.к. использует расстояния).
- Плохо работает с разреженными данными.

---

#### Линейные модели для регрессии

Общая формула: `ŷ = w[0] * x[0] + w[1] * x[1] + ... + w[p] * x[p] + b`
- `w` – веса (коэффициенты), `b` – свободный член (intercept).
- В `scikit-learn` веса хранятся в атрибуте `coef_`, `b` – в `intercept_`.

**1. Линейная регрессия (OLS) – `LinearRegression`**
- Минимизирует сумму квадратов отклонений (MSE) на обучающем наборе.
- Проста, но не имеет регуляризации → склонна к переобучению при большом числе признаков.
- Пример: на расширенном наборе **Boston** (данные о ценах на жильё в Бостоне из `sklearn.datasets.load_boston` – 506 образцов, 13 признаков; авторы книги также используют расширенную версию `load_extended_boston` из `mglearn.datasets` с добавлением полиномиальных и интерактивных признаков, всего 105 признаков) R² на обучении = 0.95, на тесте = 0.61 – явное переобучение.

**2. Гребневая регрессия (Ridge) – `Ridge`**
- Добавляет L2-регуляризацию: штрафует большие коэффициенты (их сумма квадратов).
- Целевая функция: минимизировать `MSE + alpha * (w[0]^2 + ...)`.
- Параметр `alpha` – сила регуляризации. Чем больше `alpha`, тем сильнее сжатие весов к нулю, проще модель.
- На том же Boston: `Ridge(alpha=1.0)` даёт R² на обучении 0.89, на тесте 0.75 – лучше обобщает, чем OLS.
- При `alpha=0.1` R² на тесте = 0.77, при `alpha=10` – 0.64.
- Важно: при увеличении объёма данных регуляризация становится менее критичной.

**3. Лассо (Lasso) – `Lasso`**
- Использует L1-регуляризацию: штраф – сумма абсолютных значений коэффициентов.
- Приводит к тому, что многие коэффициенты становятся точно равными нулю – автоматический отбор признаков.
- Параметр `alpha` – аналогично.
- На Boston: `alpha=1` – использует только 4 признака, R² на тесте 0.21 (недообучение). `alpha=0.01` – использует 33 признака, R² на тесте 0.77. `alpha=0.0001` – использует 94 признака, R² на тесте 0.64 (переобучение).

**Выбор между Ridge и Lasso**:
- Ridge лучше, если важны все признаки (или заранее неизвестно, какие важны).
- Lasso – если подозреваем, что важны только некоторые признаки, или нужна интерпретация.
- `ElasticNet` – комбинация L1 и L2, даёт лучшее качество, но имеет два параметра.

#### Линейные модели для классификации

Общая идея: прогноз – знак линейной комбинации. Граница решений – гиперплоскость.

**Две основные модели**:
1. **Логистическая регрессия (`LogisticRegression`)** – оценивает вероятности классов.
2. **Линейный SVM (`LinearSVC`)** – строит разделяющую гиперплоскость с максимальным зазором.

**Параметр регуляризации** `C` – обратная сила регуляризации.
- Чем больше `C`, тем меньше регуляризации, модель пытается правильно классифицировать все обучающие точки (может переобучаться).
- Чем меньше `C`, тем сильнее регуляризация, веса сжимаются, модель проще.

**Пример на датасете `cancer`** (описание выше):
- `LogisticRegression(C=1)` – accuracy на обучении 0.953, на тесте 0.958.
- `C=100` – accuracy на обучении 0.972, на тесте 0.965 (лучше).
- `C=0.01` – accuracy на обучении 0.934, на тесте 0.930 (хуже).
- Коэффициенты сильно меняются при изменении `C`; знаки могут меняться.

**L1-регуляризация** (`penalty='l1'`) даёт разреженные модели – обнуляет неважные признаки, упрощает интерпретацию.

**Мультиклассовая классификация** – стратегия «один против остальных» (one-vs-rest). Для каждого класса строится бинарный классификатор. Прогноз – класс с максимальным значением решающей функции.

---

#### Наивные байесовские классификаторы

- Основаны на теореме Байеса и предположении о независимости признаков.
- Очень быстрые, требуют мало данных, хорошо работают с высокоразмерными разреженными данными (текст).
- **Виды**:
  - `GaussianNB` – для непрерывных признаков (считает, что они имеют нормальное распределение).
  - `BernoulliNB` – для бинарных признаков (0/1).
  - `MultinomialNB` – для счётных признаков (например, частоты слов).
- Параметр `alpha` – сглаживание (чем больше, тем проще модель).
- Часто используются как baseline для классификации текстов.

---

#### Деревья решений

**Принцип**:
- Рекурсивное разбиение данных по одному признаку и порогу, чтобы максимально уменьшить неопределённость (энтропию или индекс Джини).
- Процесс продолжается, пока листья не станут «чистыми» (содержат один класс) или не сработает критерий остановки.

**Контроль сложности (предварительная обрезка)**:
- `max_depth` – максимальная глубина дерева.
- `min_samples_split` – минимальное число объектов в узле для разбиения.
- `min_samples_leaf` – минимальное число объектов в листе.
- `max_leaf_nodes` – максимальное число листьев.

**Пример на `cancer`** (описание выше):
- Дерево без ограничений даёт 100% точность на обучении, 0.937 на тесте (переобучение).
- При `max_depth=4` – точность на обучении 0.988, на тесте 0.951 (лучше).

**Анализ модели**:
- Визуализация дерева с помощью `export_graphviz` (можно посмотреть путь решений).
- Важность признаков (`feature_importances_`) – суммарное уменьшение неопределённости при разбиениях по данному признаку. Сумма важностей = 1.

**Особенности**:
- Не требуют масштабирования.
- Интерпретируемы (для небольших деревьев).
- Не умеют экстраполировать – прогноз за пределами диапазона обучающих данных будет равен последнему значению (например, в задаче временных рядов).

**Демонстрационный набор** – **two_moons** (синтетический набор из `sklearn.datasets.make_moons` – два пересекающихся полумесяца, задача бинарной классификации, часто используется для иллюстрации нелинейных границ).

---

#### Ансамбли деревьев: случайный лес и градиентный бустинг

**Случайный лес (`RandomForestClassifier`, `RandomForestRegressor`)**:
- Строит много деревьев (параметр `n_estimators`) на разных подвыборках данных.
- **Случайность**:
  1. Бутстреп-выборка – каждый раз выбирается `n_samples` объектов с возвращением (около 63% уникальных объектов попадают в выборку).
  2. При каждом разбиении рассматривается случайное подмножество признаков (параметр `max_features`).
- Прогноз: усреднение (регрессия) или голосование по вероятностям (классификация).
- **Параметры**:
  - `n_estimators` – чем больше, тем лучше, но дольше и больше памяти.
  - `max_features` – обычно `sqrt(n_features)` для классификации, `n_features` для регрессии.
  - `n_jobs` – число ядер для параллелизации.
- **Важность признаков** – более надёжна, чем у одного дерева, усредняется по всем деревьям.
- На `cancer` случайный лес со 100 деревьями даёт точность на тесте 0.972 (без настройки).
- **Преимущества**: мощный, устойчивый, не требует масштабирования, хорошо работает на больших данных.
- **Недостатки**: не интерпретируем, плохо работает с разреженными данными, требует больше памяти и времени, чем линейные модели.

**Градиентный бустинг деревьев (`GradientBoostingClassifier`, `GradientBoostingRegressor`)**:
- Строит деревья последовательно: каждое следующее дерево обучается на ошибках предыдущего (градиентном спуске).
- Используются неглубокие деревья (обычно `max_depth=1–5`), так называемые «слабые ученики».
- **Параметры**:
  - `n_estimators` – число деревьев. Увеличение может привести к переобучению.
  - `learning_rate` – скорость обучения (вклад каждого дерева). Маленькая `lr` требует больше деревьев, но даёт лучшую обобщающую способность.
  - `max_depth` – обычно 3–5.
- На практике часто показывает лучшие результаты, чем случайный лес, на табличных данных.
- **Преимущества**: высокая точность, быстрее предсказывает (чем случайный лес), меньше памяти.
- **Недостатки**: чувствителен к настройке, дольше обучается, склонен к переобучению при большом числе деревьев.
- Популярная реализация – **XGBoost** (не входит в `scikit-learn`).

---

#### Ядерный метод опорных векторов (SVM)

- Расширение линейного SVM с помощью ядерного трюка.
- Позволяет строить нелинейные разделяющие поверхности, добавляя нелинейные признаки без явного их вычисления.
- Наиболее популярное ядро – **RBF (гауссовское)**.
- Формула ядра: `k(x1, x2) = exp(-gamma * ||x1 - x2||^2)`.
- **Параметры**:
  - `C` – регуляризация (аналогично линейным моделям). Большое `C` – стремится правильно классифицировать все точки, может переобучаться.
  - `gamma` – определяет радиус влияния одной точки. Малое `gamma` – далёкие точки влияют сильно → гладкая граница (простая модель). Большое `gamma` – влияние только ближайших точек → сложная, «изрезанная» граница.
- **Настройка**: `C` и `gamma` подбираются совместно (обычно по логарифмической сетке).
- **Критически важно**: **масштабирование данных**! Без него SVM работает ужасно. На датасете `cancer` (описание выше) без масштабирования точность 63%, с `MinMaxScaler` – 97%.
- **Преимущества**: мощный, работает как с малым, так и с большим числом признаков, даёт сложные границы.
- **Недостатки**: медленный на больших данных (O(n_samples^2) или O(n_samples^3)), сложная интерпретация, требует тонкой настройки.

---

#### Нейронные сети (многослойный персептрон, MLP)

- `MLPClassifier` и `MLPRegressor`.
- Структура: входной слой → один или несколько скрытых слоёв → выходной слой.
- Каждый нейрон вычисляет взвешенную сумму входов и применяет нелинейную функцию активации (ReLU, tanh, sigmoid).
- **Параметры**:
  - `hidden_layer_sizes` – кортеж с числом нейронов в каждом скрытом слое.
  - `activation` – функция активации (`'relu'`, `'tanh'`, `'logistic'`).
  - `alpha` – L2-регуляризация (штраф на веса).
  - `solver` – алгоритм оптимизации (`'adam'` – по умолчанию, `'lbfgs'` – для небольших данных, `'sgd'` – стохастический градиентный спуск).
  - `max_iter` – максимальное число итераций.
- **Обучение**: веса инициализируются случайно, поэтому результат может меняться при разных `random_state`.
- **Масштабирование**: критически важно – нужно привести признаки к нулевому среднему и единичной дисперсии (`StandardScaler`). На `cancer` без масштабирования точность 0.92, с масштабированием и `max_iter=1000` – 0.965.
- **Регуляризация**: увеличение `alpha` помогает бороться с переобучением.
- **Преимущества**: могут моделировать очень сложные зависимости, особенно эффективны на больших данных.
- **Недостатки**: требуют много данных и времени, чувствительны к настройке, «чёрный ящик».

---

### Оценка неопределённости прогнозов (decision_function, predict_proba)

- **`decision_function`** – возвращает расстояние до разделяющей гиперплоскости (для SVM, линейных моделей). Знак определяет класс, модуль – уверенность.
- **`predict_proba`** – возвращает вероятности принадлежности к классам (для логистической регрессии, градиентного бустинга, нейросетей).
- Для мультиклассовой классификации: `decision_function` и `predict_proba` имеют форму `(n_samples, n_classes)`.
- Важно: вероятности могут быть плохо калиброваны, особенно у переобученных моделей.

---

### Выводы и перспективы главы 2

- **Сложность модели** – ключевое понятие. Баланс между недообучением и переобучением.
- **Рекомендации по выбору алгоритма**:
  - Начать с простых: линейные модели, k-NN, наивный Байес.
  - Затем переходить к более сложным: деревья, случайный лес, бустинг, SVM, нейросети.
- **Важность масштабирования** – для k-NN, линейных моделей, SVM, нейросетей.
- **Интерпретация** – деревья и линейные модели проще объяснить, чем ансамбли или нейросети.
- **Данные** – больше данных обычно важнее, чем тонкая настройка алгоритма.

## Глава 3 (начало). Обучение без учителя и предварительная обработка

### Типы задач без учителя

1. **Преобразование данных** – изменение представления данных для упрощения визуализации, сжатия, выделения признаков.
2. **Кластеризация** – группировка схожих объектов.

**Проблема**: отсутствие правильных ответов → сложность оценки качества (часто требуется ручной анализ).

### Предварительная обработка и масштабирование

- Многие алгоритмы (особенно SVM, нейросети, k-NN) чувствительны к масштабу признаков.
- Признаки с большим разбросом могут доминировать и искажать результат.

**Методы масштабирования в `sklearn.preprocessing`**:

| Метод | Описание | Когда использовать |
|-------|----------|-------------------|
| `StandardScaler` | Стандартизация: `(x - mean) / std` для каждого признака. Среднее 0, дисперсия 1. | Чаще всего для SVM, нейросетей, линейных моделей. |
| `MinMaxScaler` | Нормализация по диапазону: `(x - min) / (max - min)` → диапазон [0, 1] (или другой). | Когда нужно строго ограничить диапазон; устойчив к выбросам. |
| `RobustScaler` | Использует медиану и квартили: `(x - median) / (Q3 - Q1)`. Устойчив к сильным выбросам. | Когда в данных много выбросов. |
| `Normalizer` | Нормализует каждую **строку** (объект) к единичной евклидовой длине. | Когда важен угол вектора (например, в текстовых данных, где длина не имеет значения). |

**Правила применения fit / transform / fit_transform**:
1. **`fit`** – вычисляет параметры масштабирования **только на обучающем наборе** (`scaler.fit(X_train)`).
2. **`transform`** – применяет преобразование к любому набору, используя параметры из обучения (`scaler.transform(X_train)` и `scaler.transform(X_test)`).
3. **`fit_transform`** – удобное сокращение для `fit` + `transform` на одном наборе (обычно на обучающем).

**Ошибка**: **Никогда не вызывать `fit` на тестовом наборе!** Это приведёт к утечке данных и переоценке качества модели. Тестовые данные должны быть преобразованы только с параметрами, полученными из обучения. Иллюстрация: если масштабировать обучающий и тестовый наборы по отдельности, их распределения могут сместиться, и тестовые точки окажутся в другой системе координат.

**Влияние масштабирования на качество моделей (пример с SVM)**:
- На датасете `cancer` (описан выше) без масштабирования SVM (C=100) даёт точность на тесте 0.63.
- С `MinMaxScaler` – точность поднимается до 0.97 (эффект колоссальный).
- С `StandardScaler` – точность ≈ 0.96.

**Вывод**: масштабирование – обязательный шаг для многих алгоритмов; его нельзя игнорировать.
